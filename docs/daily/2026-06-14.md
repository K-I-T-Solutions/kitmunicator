# Daily Report – 2026-06-14

## Durchgeführte Arbeiten

### Bug: Eingehende Anrufbenachrichtigung funktionierte nicht

**Symptom:** Browser von Jessica zeigte kein Incoming-Call-Overlay, wenn Joshua anrief.

**Root Cause:** Asterisk verwendete den Docker-Container-Hostnamen (`79de6809ad35`) als Domain in ausgehenden SIP `From`-Headern:

```
From: "Joshua Phu Kuhrau" <sip:1001@79de6809ad35>;tag=...
```

SIP.js lehnt Hostnamen ohne Punkt als ungültige SIP-URI ab und dropt das INVITE-Paket stillschweigend (`error parsing header 'From'`). Der `onInvite`-Callback wurde deshalb nie aufgerufen.

**Fix:** `from_domain = kitmunicator.intern.phudevelopement.xyz` im `[webrtc_endpoint](!)` Template in `pjsip.conf` gesetzt. Asterisk setzt nun eine gültige FQDN in den From-Header:

```
From: "Joshua Phu Kuhrau" <sip:1001@kitmunicator.intern.phudevelopement.xyz>;tag=...
```
